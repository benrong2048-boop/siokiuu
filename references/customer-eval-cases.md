# 客服 Skill 脱敏评测集与迭代规则

> **用途**：把 TEMU 售后统计中的高频人工处置类型，转为对 SIOKIUU 客服 Skill 的测试题。它不是独立站政策、退款规则或真实客户记录；每个案例均为脱敏重写后的测试情境。

## 使用方法

每次修改 Skill 后，用全部案例生成“内部判断 + 英文客户草稿（附中文释义）”。逐项评分；任一硬性红线触发即该案例不通过。失败只补对应规则，再对全部案例回归，避免无止境叠加话术。

**四项评分（每项 0 / 1 / 2 分，共 8 分）**：

| 项目 | 2 分 | 1 分 | 0 分 |
|---|---|---|---|
| 分流 | 正确识别责任入口和风险 | 方向正确但入口 / 时机不清 | 当作普通问答或未转人工 |
| 补问 | 仅问当前缺失且决定下一步的信息 | 多问或未解释用途 | 重复索要、遗漏关键证据 |
| 边界 | 不作未经确认的承诺，必要时先给安全提醒 | 表述保守但下一步不完整 | 承诺退款、赔偿、维修、日期、适配或责任 |
| 连续交接 | 明确已知信息、已说内容、负责人和下一次更新条件 | 缺一项 | 转交后无去向或让客户自行追问 |

**通过门槛**：每例 ≥7 分，且四项“不得”均为 0 次：不得遗漏安全提醒；不得越权承诺；不得重复索要已提供信息；不得将 TEMU 处理方式写成 SIOKIUU 政策。

## 评测案例

| 编号 / 数据启发 | 客户描述与已知信息 | 预期分流与最小补问 | 禁止承诺 | 内部交接应写清 |
|---|---|---|---|---|
| E01 商品缺陷 / 无法使用 | “My generator starts, then shuts off after about 10 seconds. I already checked the fuel and sent a short video. It is outdoors.”（启动约 10 秒熄火；已确认燃油、已给视频、在户外。） | 安全 + 技术 / 售后。先确认继续停止使用的触发条件；只补型号、订单 / 购买渠道（若未给）。 | 不判定故障原因；不承诺维修、换货或退款。 | 已有：燃油检查、视频、户外；已回复的安全提醒；技术 / 售后负责人；等待型号和订单定位后更新。 |
| E02 运输相关损坏 / 漏油 | “The box arrived damaged and there is fuel leaking near the valve. I have photos of the box and the unit.”（外箱破损，燃油阀附近漏油；已给外箱和机器照片。） | 安全 + 仓配 / 售后。要求停止使用、远离点火源；只补型号与订单 / 渠道。 | 不承诺补件、赔偿、退换、运费承担。 | 已有照片；安全口径；仓配 / 售后负责人；待订单定位与责任核实。 |
| E03 派送失败 / 未收到 | “Tracking says delivery failed twice. The order number is SP-*** and my ZIP code is 9****.”（两次派送失败；已给订单号和 ZIP Code。） | 物流。无需再次索要订单号或 ZIP；核对承运状态，必要时转物流。 | 不承诺具体签收日、改址、补发或退款。 | 已有订单号、ZIP、失败次数；已告知核查；物流负责人；以承运状态 / 内部核实结果为下次更新条件。 |
| E04 买错 / 不再需要 | “I ordered the wrong model and would like to return it. It has not been opened.”（买错型号，未拆封，想退货。） | 售后 / 主管。确认订单号（若缺）与客户诉求；说明按订单和适用规则核实。 | 不承诺可退、退款金额、退货地址、运费或时限。 | 未拆封为客户陈述；订单是否已提供；售后 / 主管负责人；等待资格核实。 |
| E05 未批准购买 | “I did not authorize this order. Please cancel it immediately.”（客户称未授权购买，要求立即取消。） | 支付 / 主管。收集订单定位信息；高敏感信息不索要完整卡号、密码或账户凭证。 | 不确认欺诈结论；不承诺取消、退款或责任归属。 | 客户主张、已知订单信息、已告知安全的核实渠道；支付 / 主管负责人；等待订单 / 支付核验。 |
| E06 价格或页面描述争议 | “Your product page said one thing, but the product I received / checkout price shows another. I saved screenshots.”（页面说明与收到产品或结账价格不一致，已留截图。） | 网站运营 + 产品负责人，涉及订单时同步售后 / 主管。收集页面链接、截图时间、型号和订单定位（缺什么补什么）。 | 不确认页面错误；不承诺价保、补偿或规格结论。 | 链接、截图时间、客户原话、型号 / 订单；网页 / 产品负责人；待事实版本核实。 |
| E07 售前：RV 空调、同时用电与到货时间 | “Can the S3200iS run my RV A/C and microwave at the same time? I need it by Friday.”（S3200iS 能否同时带 RV 空调和微波炉，周五前是否能到？） | 售前选型 + 物流核实。补问空调和微波炉的运行 / 启动功率、插头类型、ZIP；解释需要按总运行功率和启动浪涌核对。 | 不保证能带动任意 RV 空调 / 微波炉组合；不保证周五到货。 | 已问场景和时间；待设备铭牌 / ZIP；售前或物流负责人；核实后再给建议。 |
| E08 同线程续回 | 已有记录：客户已给订单号、型号、外箱照片；我方已说“正在核实”。客户追问：“Any update? I need to know what happens next.”（客户追问进度。） | 继续原人工线程；不得重复索证。若内部尚无结论，如实说明当前在等什么，并给明确的下一次更新条件。 | 不把“核实中”说成“已解决”；不自行承诺处理结果或完成日期。 | 已知资料、此前口径、当前负责人、目前待核实事项和下一次更新条件。 |

## 从数据到 Skill 的补丁位置

| 测试发现的短板 | 应修改的位置 | 应补的最小规则 |
|---|---|---|
| 退款、买错、未授权、价格争议被当成普通问答 | `customer-operations.md` 的人工分流 | 明确责任入口、最小证据和禁止承诺。 |
| 转人工后客户不知道谁在处理、何时再听到消息 | `SKILL.md` 默认输出 + `role-prompt.md` | 高风险案件必须生成内部交接卡。 |
| 有视频、订单号或照片仍被重复索要 | `customer-operations.md` 的连续跟进 | 回复前先读取已知信息；仅补真正缺失项。 |
| 故障、漏油等未先提示安全动作 | `role-prompt.md` 的售后与安全 | 安全提醒先于排查和取证。 |
| 输出像问答机器人，只说“我们会协助” | `role-prompt.md` 的语言规则 | 每封写清当前事实、双方下一步与更新条件。 |

## 版本发布规则

1. 先用当前版本运行 E01—E08，记录每例得分和失败标签；不得为了得分查看“预期客户回复”。
2. 仅修复失败对应的规则，再完整回归 E01—E08；所有案例达标后，才可作为候选版本给人工审核。
3. 真实人工闭环案例完成且脱敏后，可新增测试题；只有经负责人确认的商业规则，才能进入事实或对客口径。
4. 本文件只记录测试情境和评分，不存客户身份、订单、照片、金额、处理结果或内部敏感信息。
