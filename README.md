# SIOKIUU 客户全旅程协同 Skill

这是一个供 SIOKIUU Power 团队试用的 Codex Skill。它不是自动回复机器人；它帮助使用者基于已整理的产品、网站、说明书、工厂与匿名客户信号，完成**有依据、有边界、让客户感到安心**的客户沟通与运营判断。

适用范围：售前咨询、型号选择、信任建立、配送/政策说明、使用咨询、故障售后、FAQ 与页面优化建议、营销转化文案。

> 面向美国客户的最终回复默认使用英文。内部审阅版会在英文后附中文释义；明确要求“可直接发送”时，只输出英文。

| 帮客户在型号间选择 | `$siokiuu 客户需要给 RV 的日常小电器供电，重视安静和便携。请说明需要补问什么，并给出不越界的选型回复。` |
| 处理售后异常 | `$siokiuu 客户说设备启动约10秒后熄火。请给出安全优先的处理建议和英文邮件草稿。` |
| 优化 FAQ 或页面 | `$siokiuu 根据客户反复询问的配送时效，判断现有 FAQ 是否足够，并给出可审核的英文 FAQ 文案。` |
| 形成营销内容 | `$siokiuu 面向周末露营和轻量 RV 用户，为 S2500i 写一段有购买动作的英文落地页文案；只使用已确认事实。` |

## 工作方式

```text
客户问题 / 运营任务
        ↓
识别真实处境与风险
        ↓
按任务读取对应事实资料
        ↓
核对冲突与承诺边界
        ↓
给出行动建议 + 英文草稿 + 下一步
```

客服场景固定遵循：**理解处境 → 给出方向 → 解释必要事实 → 明确客户与公司的下一步**。

售前场景固定遵循：**场景 → 使用任务 → 已确认价值 → 证据 → 消除阻碍 → 购买动作**。

## 资料结构与可信边界

| 资料 | 用途 | 可否直接作为对客结论 |
| --- | --- | --- |
| `references/website-facts.md` | 当前公开产品页、FAQ、政策与场景页 | 可以，但只代表当前网页展示 |
| `references/manual-facts.md` | 安全、启动、维护、储存与排故步骤 | 可以，用于对应型号的操作指引 |
| `references/product-facts.md` | 型号、功率、接口、场景与已知冲突 | 可以；冲突项须标记“待确认” |
| `references/factory-notes.md` | 负载、保护与技术判断的内部背景 | 不单独作为对客承诺依据 |
| `references/customer-signals.md` | 匿名高频问题与历史客户信号 | 用于判断优先级与 FAQ 改进，不还原个人信息 |
| `references/customer-operations.md` | 语气、转人工、冲突处理与沟通边界 | 用于组织回复和内部动作 |
| `references/role-prompt.md` | 完整角色提示词与服务标准 | 每次使用前优先读取 |

### 已知待确认项

资料中存在型号命名、部分功率、并机效率、启动方式、配送时效与售后费用等不一致。Skill 不会自行选择“更好看”的版本。涉及这些内容时，必须使用保守表达并转交负责人确认。

## 使用与安装

### 在 GitHub 上查看

直接打开 `SKILL.md` 阅读处理规则，按问题进入 `references/` 查证具体资料。无需向仓库提交客户邮件、订单信息或账号凭证。

### 在 Claude Code 中本地试用（推荐）

1. 下载本仓库的 ZIP，或执行 `git clone https://github.com/benrong2048-boop/siokiuu.git`。
2. 将整个 `siokiuu` 文件夹放入 Claude Code 的个人 Skills 目录：`C:\Users\你的用户名\.claude\skills\siokiuu`。
3. 新开一个 Claude Code 会话。
4. 在对话中输入 `$siokiuu` 加上实际业务问题。

不要只复制 `SKILL.md`，`references/` 必须一并保留。若已有项目级 `.claude/skills/` 目录，也可以将整个文件夹放在项目目录下供该项目使用。

### 在 Codex 中本地试用

将整个 `siokiuu` 文件夹放入 `C:\Users\你的用户名\.codex\skills\siokiuu`，新开会话后以 `$siokiuu` 调用；其余要求与上面相同。

## 首轮试用：5 个验收问题

建议由上级或同事按下列情境测试，并记录“事实是否准确、是否有边界、是否让客户知道下一步”。

1. **品牌信任**：`你们这个发电机靠谱吗，是不是骗人的？`
2. **型号选择**：`3200W 的好用吗？我想给 RV 用。`
3. **安全售后**：`设备运行约 10 秒后熄火，怎么处理？`
4. **政策问题**：`什么时候能送到？如果不合适能退吗？`
5. **营销转化**：`为安静露营场景写一段 S2500i 的购买引导文案。`

通过标准不是“回答得长”，而是：

- 只使用资料可支持的事实；
- 对风险和冲突不强行下结论；
- 英文自然、尊重客户处境，且不生硬推销；
- 客户和内部人员都能知道下一步该做什么。

## 不应使用本 Skill 的情况

- 直接承诺退款、换货、赔付、确切签收日期、维修结论或未确认的产品适配性；
- 处理客户完整支付信息、密码、账号凭证或未脱敏订单数据；
- 用未经授权的外部来源补全产品事实；
- 要求自动向客户发送回复。

以上情况应转交对应负责人；Skill 只可输出待审核草稿和所需确认信息。

## 版本与反馈

当前版本为试用版。测试中如发现“事实缺失、表述不安心、转人工不清楚或资料冲突”，请在 GitHub Issue 中写明：原始问题（脱敏）、Skill 输出、问题所在和建议改法。不要上传客户原邮件或个人信息。
